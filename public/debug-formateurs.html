Test Formateurs Debug

Debug Instructions

  1. Navigate to the admin page (formateurs section)
  2. Open the browser console (F12)
  3. Look for the detailed debug logs when the page loads
  4. Try clicking a delete button on any formateur
  5. Check the comprehensive debug logs in the console
  6. You can also run these debug commands in console:

Debug Commands:

  • window.inspectFormateursData() - Inspect current data state
  • window.manualDeleteTest(1) - Test delete with specific ID
  • window.reloadFormateursData() - Reload data from API
  • allFormateurs - View all formateurs array
  • filteredFormateurs - View filtered formateurs array

Direct API Testing

Test the API endpoints directly:

// Test GET endpoint

fetch('/admin/api/formateurs').then(r => r.json()).then(console.log)

// Test DELETE endpoint (replace 1 with actual ID)

fetch('/admin/api/formateurs/1', {
  method: 'DELETE',
  headers: {
    'X-CSRF-TOKEN': document.querySelector('meta[name="csrf-token"]').getAttribute('content')
  }
}).then(r => r.json()).then(console.log)

Expected Debug Output

When you click a delete button, you should see detailed logs showing:

  • The formateur ID being passed to deleteFormateur()
  • The current state of allFormateurs array
  • Multiple comparison methods being tested
  • Whether the formateur is found with each method
  • The deletion process step by step

Note: If the delete still doesn't work, the debug logs will show exactly why the formateur cannot be found, which will help us identify the root cause.
